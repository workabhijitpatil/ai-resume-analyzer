UI States
Empty state
Both panels show placeholder text. Button is disabled and grayed out. Upload area has dashed border with upload icon.
Loading state
Button shows animated shimmer: "✦ Optimizing..." — output panel shows a skeleton loader (3 gray lines, pulsing). User cannot click again.
Success state
Output panel fills with rewritten resume. Copy + Download buttons appear. A small teal badge: "Optimized for this JD" appears at the top.
Error — wrong file type
Red inline message below upload: "Only PDF and DOCX files are accepted." Upload box shakes briefly (CSS animation).
Error — API failure
Toast notification (top-right): "Something went wrong. Try again." — red border, auto-dismisses in 4s.
Mobile state
Two-column layout stacks vertically. Input panel on top, output panel below. Button stays full-width. Nav collapses to logo only.
Signature Element — The "Optimize" Button
The single memorable moment in this UI. While the AI is working, the button switches from solid teal to an animated shimmer — a slow left-to-right gradient sweep — with the label changing to "✦ Optimizing your resume...". This is the only animation in the entire app, which makes it land harder.
