PRD — Product Requirements Document

Overview
An AI-powered web app where job seekers (especially freshers) upload their resume as a PDF/DOCX, paste a job description, and receive an AI-rewritten resume optimized for that specific JD — with better keywords, improved phrasing, and ATS-friendly formatting. No job-matching engine in v1; the AI does the heavy lifting.
Core MVP Features
  • Resume upload — accept PDF and DOCX files, extract raw text from them
  • JD input — a textarea where the user pastes the full job description
  • AI rewrite — Claude API reads both and returns an optimized, rewritten resume
  • Result display — show the rewritten resume in a clean, readable format
  • Copy/download — let the user copy or download the result as plain text
User Flow
  1. User lands on homepage — sees a clean upload box and a textarea
  2. User uploads their resume (PDF or DOCX)
  3. User pastes the job description into the textarea
  4. User clicks "Optimize Resume" — a loading state appears
  5. AI rewrites the resume, result renders on the same page
  6. User copies or downloads the optimized resume
Business Logic & Rules
  • Both resume and JD must be provided before submitting
  • Max file size: 5MB
  • Only PDF/DOCX accepted — reject other formats with a clear error
  • AI must preserve factual accuracy — no invented experiences
  • Show a loading spinner during API call (can take 5–15 sec)
Out of Scope (v1)
  • Job matching / job board scraping
  • User accounts / login
  • Resume score / ATS score display
  • Save / history of past resumes
  • PDF export of the result

Why this stack?
FastAPI handles file uploads cleanly in Python, which is also where your PDF/DOCX parsing libraries live. React + Vite gives a fast, modern frontend. Claude API is the core AI brain. No database needed in v1 — everything is stateless per request.

Risk Areas to Watch

CORS errors between React frontend and FastAPI backend — configure CORS headers on the backend from day 1
PDF text extraction fails on scanned/image-based resumes — show a clear error message, don't silently fail
Claude API response time is slow (5–15s) — always show a loading spinner, never leave the user staring at a blank screen
API key exposed in frontend — NEVER put the Claude API key in React code, always call it through FastAPI backend
Claude hallucinating new experiences — add a strict prompt instruction: "only rephrase and restructure, never invent"

# AI Resume Analyzer — Cursor Rules

## Stack
- Frontend: React + Vite + Tailwind CSS
- Backend: Python FastAPI
- AI: Claude API via backend only

## Code Style
- Use TypeScript for all React files
- Functional components only, use React hooks
- Tailwind for ALL styling — no inline styles, no CSS files
- Python: PEP8 style, type hints on all functions

## Security Rules
- NEVER put the Claude API key in any frontend file
- All Claude API calls go through the FastAPI backend
- Validate file type and size on both frontend and backend

## Behavior Rules
- Make minimal changes — don't refactor unrelated code
- Always explain what you changed and why
- No TODOs left in code — everything must work
- Show loading states for any async operation
- Handle errors with user-friendly messages, not raw stack traces
